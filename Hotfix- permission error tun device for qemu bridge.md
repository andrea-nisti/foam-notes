# Hotfix: permission error tun device for qemu bridge

#qnx #qemu #hotfix #networking

[//begin]: # "Autogenerated link references for markdown compatibility"

When launcing  `mkqnximage --type=qemu --run --hostname=qnx` from command line or when creating a new qemu target in vscode, the following error occurs:

    failed to create tun device: Operation not permitted qemu-system-x86_64: -netdev bridge,br=br0,id=net0: bridge helper failed

## Solution

```bash
# Set capabilities (preferred over setuid)
sudo setcap cap_net_admin+ep <path_to_bridge>/qemu-bridge-helper

# Create the ACL file to allow bridge access
sudo mkdir -p /etc/qemu
echo "allow br0" | sudo tee /etc/qemu/bridge.conf
```
